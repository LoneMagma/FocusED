# Code of Conduct

## The short version

Be direct. Be respectful. Assume good intent.  
This is a small project — we don't need a committee, just basic decency.

---

## Standards

**Expected:**
- Constructive technical disagreement ("this approach will cause X because Y")
- Honest feedback on PRs, even critical feedback
- Respecting that maintainers will decline changes that don't fit the project's design

**Not acceptable:**
- Personal attacks or insults
- Harassment of any kind
- Dismissing contributions without explanation
- Spamming issues or PRs

---

## Enforcement

If someone is being a problem, email the maintainer directly.  
Persistent issues result in being blocked from the repository.

This project is maintained in free time. We will not entertain lengthy disputes about enforcement decisions.

---

## Scope

This applies to the GitHub repository: issues, PRs, discussions, and code review comments.
